# Текущая задача: Создание тестового окружения в Docker

## Зачем
Для упрощения тестирования проекта на разных "устройствах" без необходимости иметь реальное железо. Это позволяет имитировать кластер из одной мастер-ноды и двух агентов.

## Что сделано
1.  **Модификация r4a-server**: Добавлена поддержка переменной окружения `R4A_SECRET`. Если она установлена, сервер использует её как секрет кластера. Это необходимо для автоматической связи агентов с мастером в Docker Compose.
2.  **Dockerfile.test**: Создан многоэтапный Dockerfile.
    *   **builder**: Компилирует `r4a-server` и `r4a-agent` в среде Rust 1.77.
    *   **runtime**: Легковесный образ на базе Debian с установленными `wireguard-tools`, `iproute2`, `iptables` и `docker.io`.
3.  **compose.yaml**:
    *   Поднимает 3 контейнера: `r4a-master`, `r4a-agent1`, `r4a-agent2`.
    *   Ограничение ресурсов: 1 ядро CPU и 1 ГБ RAM для каждого контейнера.
    *   Использование `cap_add: [NET_ADMIN]` и проброс `/dev/net/tun` для работы WireGuard.
    *   Проброс `/var/run/docker.sock` для агентов (чтобы они могли управлять контейнерами).
    *   Использование фиксированного `R4A_SECRET` для автоматического объединения в кластер.

## Как проверить
1.  Убедиться, что Docker и Docker Compose установлены.
2.  Запустить кластер:
    ```bash
    make dev-up
    ```
3.  Развернуть обновления (после правок кода):
    ```bash
    make dev-deploy
    ```
4.  Проверить логи, пинги или запустить TUI:
    ```bash
    docker exec node-agent1 ping 10.42.0.1
    docker exec -it node-agent1 r4a-tui
    ```
